今日は株式会社ヘンリー主催のイベントに参加してきました！忘れないうちに、参加して良かったことや感じたことを書き留めておきます。

[https://henry.connpass.com/event/381632/:embed:cite]

## 登壇内容の概要

登壇ではヘンリーが Google Cloud から AWS への移行を決めた背景についてお話をされていました。

当時はインフラ専任の方がいない中で、スピード感を持ってサービスを立ち上げるために Google Cloud を選んだのは大正解だったそうです。

そこから組織が大きくなり、病院向けにサービスを広げていく中で、「もっと高い信頼性や運用しやすさが必要だ」というフェーズに。アプリとの相性なども考慮した結果、より細かくコントロールできる AWS への移行を決めた、というストーリーでした。

詳しい内容はエンジニアブログにも載っているので、気になる方はぜひ！

[https://dev.henry.jp/entry/2025/why-we-are-migrating-from-google-cloud-to-aws:embed:cite]

## 感想

個人的になるほどなと思ったのが、クリニックと病院の違いについてのお話です。病院向けサービスはクリニックと異なり「入院ワークフロー」が必要不可欠となります。そこでは医師や看護師、薬剤師、検査技師といった多職種が連携する必要があり、クリニック向けフェーズの時よりもシステムの複雑性が増すとのことです。扱う業務範囲が格段に広がる分、システム側にもそれに応える高度な設計が求められるということですね。

また、入院患者さんは 24 時間いらっしゃるため、一瞬の停止も許されない「24時間365日の信頼性」が不可欠になるというお話もありました。この「24/365 の重み」が、今回の AWS 移行という大きな決断に直結しているのだと聞き、すごく腑に落ちました。

何より、この大きな決断を「今しかない」と見極めて進めるチームの覚悟を生で聞けたのは良かったです。状況を冷静に分析して、ベストなタイミングで舵を切れるのは本当にかっこいいなと思いました。

懇親会でも、社員の皆さんが技術的な話から医療への想いまで気さくに話してくださり、あっという間の時間でした。

今日のイベントを通して、ヘンリーという会社がとても好きになりました。医療という重要な分野で、技術的な課題に真摯に向き合いながら、より良いサービスを提供しようとする姿勢が伝わってきました。これからもヘンリーの成長を応援していきたいと思います!
